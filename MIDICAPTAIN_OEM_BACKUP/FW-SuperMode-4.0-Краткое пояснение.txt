Супер-режим Конфигурация функций и объяснение ключевых слов для версии FW-SuperMD4.0
(AI translated)

Удерживая кнопку 1 или кнопку key0, включите устройство. Подключите USB-кабель к компьютеру, чтобы войти в режим настройки USB. На компьютере эмулируется соответствующий диск MIDICAPTAIN, где можно найти соответствующий файл конфигурации в supersetup.

(1) Поддерживаются 4 триггерные момента для кнопок: нажатие короткое, отпускание короткое, удержание, отпускание удержания. Соответствующие ключевые слова: short_dwX, short_upX, longX, long_upX, где X - это цифра от 1 до 9, соответствующая X-му нажатию в keytimes.

	Например: short_dw1 = [1][PC][5][-] означает отправку команды PC со значением 5 и MIDI-каналом 1 в момент первого короткого нажатия.

Совет: если нужно, чтобы короткое нажатие и удержание были полностью независимыми событиями, мы можем определить только события отпускания короткого нажатия и удержания, то есть short_upX и longX. Таким образом, перед активацией удержания не будет выхода события короткого нажатия.

(2) Для каждого триггерного момента можно определить смешанные управляющие команды, включая команды MIDI PC, CC, NT (заметка) и команды HID для клавиатуры и мыши. Это может быть как одиночная команда, так и комбинация нескольких команд, без ограничения количества.

	Формат каждой команды состоит из 4 квадратных скобок [][][][]. Приведены примеры четырех различных типов команд:

	[1][PC][8][-] - команда PC с отправкой PC8 на канал MIDI 1.
	[2][CC][10][127] - команда CC с отправкой CC10 со значением 127 на канал MIDI 2.
	[1][NT][60][127] - команда NT (заметка) с отправкой ноты 60 и уровнем громкости 127 на канал MIDI 1.
	[send][HID][-][A] - команда HID для эмуляции быстрого нажатия и отпускания клавиши A.

	Можно добавить несколько команд, разделенных пробелом, например: short_dw1 = [1][PC][8][-] [2][CC][10][127] [3][CC][20][127] [send][HID][-][A] ... Обратите внимание на формат, можно добавлять пробелы, но не переносить строки.

(3) Ключевое слово keytimes позволяет определить многократное нажатие кнопки, позволяя настроить разные управляющие команды для каждого нажатия.

	keytimes = [1] указывает, что эта кнопка имеет только одну функцию.
	keytimes - [5] указывает, что эта кнопка имеет функцию повторения 5 раз, и можно настроить 5 различных команд, максимальное количество - 9 раз.

(4) Вся конфигурация находится в каталоге supersetup, в соответствующем файле pageX.txt. Каждая страница представляет собой набор функций управления, которые можно переключать, долго нажимая на крайнюю правую кнопку, с максимальным количеством страниц - 99.

	Значение X в pageX.txt должно начинаться с 0 и быть не больше 99, сохраняя непрерывность.

(5) В каждом файле pageX.txt определены три основные части. Вначале идет [globalsetup], где задаются общие параметры устройства и каждой страницы. Обратите внимание, что эта часть будет применяться только в page0.txt после перезагрузки.

	ledbright = [30] задает яркость светодиодного кольца кнопок от 1 до 100, рекомендуется значение от 20 до 50.
	screenbright = [80] - яркость экрана от 1 до 100, рекомендуется значение от 30 до 100.
	dark_fonts = [off] - переключатель темного шрифта: off для темного фона, on для светлого фона.
	wallpaper = [wp1] - настройка фонового изображения, рекомендуется оставить wp1, можно использовать изображение формата BMP размером 240x240 и глубиной цвета 24 бита, переименовав его в wp1.
	long_press_timing = [1] - время удержания для определения длинного нажатия в секундах: 1 / 1.5 / 2 / 2.5.
	WIRELESS_2.4G = [on] - переключатель внутреннего модуля 2.4G: on / off, применяется только к версиям 10- с синим и золотым переключателями, также как и следующие параметры.
	WIRELESS_ID = [8] - идентификатор беспроводного модуля, который должен совпадать с конфигурацией MIDI MATE, по умолчанию - 8, если одновременно используются несколько беспроводных устройств, установите разные идентификаторы.
	WIRELESS_dB = [6] - настройка мощности беспроводного модуля: 0:12dBm, 1:10dBm, 2:9dBm, 3:8dBm, 4:6dBm, 5:3dBm, 6:0dBm, 7:-2dBm, 8:-5dBm, 9:-10dBm, 10:-15dBm, 11:-20dBm, 12:-25dBm, 13:-30dBm, 14:-25dBm.

(6) Затем [PAGE], где определены параметры каждой страницы.

	page_name = [HXLP] Определяет имя страницы, рекомендуется 4 заглавных буквы.

	exp1_CH = [1] MIDI-канал педали выражения 1.
	exp1_CC = [1] MIDI CC-номер педали выражения 1.

	exp2_CH = [1] MIDI-канал педали выражения 2.
	exp2_CC = [2] MIDI CC-номер педали выражения 2.

	encoder_CC = [90] MIDI CC-номер энкодера.
	encoder_NAME = [MYCC] Название функции энкодера.

	midithrough = [on] Переключатель MIDI-пропуска: вкл./выкл.

	display_number_ABC = [abc3] Выбор способа отображения PC: 123 / abc3 / abc4 / abc5 / abc8 соответствуют чистой цифровой или буквенной группировке с интервалом 3, 4, 5, 8.

	group_number = [3] Здесь также устанавливается способ группировки. Когда вышеуказанные параметры установлены в режим цифровой нумерации 123, это дополнительная настройка для интервалов между группами.

	display_pc_offset = [1] Укажите 0 или 1, чтобы отображать значение PC с 0 или с 1, чтобы согласовать отображение с различными устройствами-хостами.

	display_bank_offset = [1] Укажите 0 или 1, чтобы указать, начинается ли группа с 0 или с 1 при группировке.

(7) Конкретная конфигурация кнопок, обозначенных как [key0] [key1] [key2] [key3] [key4] [key5] ... в зависимости от числа выводов, различается количество кнопок для настройки.

	Для примера рассмотрим [key0]:

	keytimes = [3] Конфигурация кнопки для трехкратного повтора.

	ledmode = [normal] Режим светодиодов кнопки установлен на нормальный, что означает, что светодиод будет гореть цветом, соответствующим номеру нажатия.

	ledmode может быть одним из: normal / select / tap. Normal - каждое последующее нажатие светится другим цветом, Select - только одна кнопка может быть выбрана в одно время, Tap - используется для меток, при этом при возврате после нажатия другой кнопки состояние кнопки будет сброшено до начального.

	ledcolor1 = [0x666666][0x666666][0xff0000] Определяет цвет первого нажатия (серый, серый, красный), где каждая пара скобок представляет цвет светодиодов в трех сегментах, префикс 0x обозначает шестнадцатеричное значение, RGB представляет значения красного, зеленого и синего цветов соответственно.

	short_dw1 = [1][CC][69][0] Определяет отправку CC-команды при первом коротком нажатии: канал 1, CC 69, значение 0.

	ledcolor2 = [0xff0000][0x666666][0x666666] Определяет цвет для второго нажатия (красный, серый, серый).
	short_dw2 = [1][CC][69][1] Определяет отправку CC-команды при втором коротком нажатии: канал 1, CC 69, значение 1.

	ledcolor3 = [0x666666][0xff0000][0x666666] Определяет цвет для третьего нажатия (серый, красный, серый).
	short_dw3 = [1][CC][69][2] Определяет отправку CC-команды при третьем коротком нажатии: канал 1, CC 69, значение 2.

(8) Другие предопределенные ключевые слова и их использование.

	Обратите внимание, что регистр важен.

	[1][PC][inc1][-] Увеличивает текущее значение PC на 1, можно использовать inc1 / inc2 / inc3 / inc4 / inc5 для увеличения на 1, 2, 3, 4 или 5.
	[1][PC][dec1][-] Уменьшает текущее значение PC на 1, можно использовать dec1 / dec2 / dec3 / dec4 / dec5 для уменьшения на 1, 2, 3, 4 или 5.
	[2][PC][random][-] Отправляет случайное значение PC на канале MIDI 2.
	[1][PC][auto][bank_inc] Позволяет автоматически инкрементировать номер банка PC.
	[1][PC][auto][bank_dec] Позволяет автоматически декрементировать номер банка PC.
	[1][PC][auto][0] Позволяет автоматически отправлять номер PC для первого звука в банке.
	[1][PC][auto][1] Позволяет автоматически отправлять номер PC для второго звука в банке.
	[1][PC][auto][2] Позволяет автоматически отправлять номер PC для третьего звука в банке.
	[1][PC][auto][3] Позволяет автоматически отправлять номер PC для четвертого звука в банке.
	[1][PC][auto][4] Позволяет автоматически отправлять номер PC для пятого звука в банке.
	[1][PC][auto][5] Позволяет автоматически отправлять номер PC для шестого звука в банке.
	[1][PC][auto][6] Позволяет автоматически отправлять номер PC для седьмого звука в банке.
	[1][PC][auto][7] Позволяет автоматически отправлять номер PC для восьмого звука в банке.

	[send][HID][-][A] Команда клавиатуры: клавиша A, быстрое нажатие и отпуск.
	[press][HID][-][A] Команда клавиатуры: клавиша A, только нажатие, без отпускания.
	[release][HID][-][A] Команда клавиатуры: клавиша A, только отпускание.
	[release][HID][-][all] Команда клавиатуры: отпускает все клавиши.
	[delay][HID][-][50] Команда задержки, 50 миллисекунд.

	[send][HID][Ctrl][A] Команда клавиатуры: составное нажатие клавиш, третья скобка может быть Ctrl / Shift / Alt / Option / Windows.

	В четвертой скобке команды для эмуляции мыши и клавиатуры могут быть следующие клавиши:

	A-Z, 0-9, F1-F12, Mouse_L (нажатие левой кнопки мыши), Mouse_R (нажатие правой кнопки мыши)

	Space, Esc, Caps, Right, Left, Up, Down, End, Del, PageUp, PageDown, Enter, Pause, Table, BackSpace, Home, Ins, PrintS (печать экрана)

(9) Советы по использованию функций.

	Команда NT note после отправки звуковой ноты может привести к продолжительному звучанию звука, необходимо добавить команду для отправки ноты с нулевой громкостью, чтобы остановить звук.
	Часто можно использовать общую команду [1][CC][123][0], чтобы выключить звук, но это зависит от поддержки основным устройством.

	Используйте [press][HID][-][A] для эмуляции нажатия клавиши, но не забудьте настроить соответствующую команду для отпускания, иначе компьютер будет продолжать реагировать на нажатие клавиши, что может привести к зависанию компьютера и другим проблемам.

	[delay][HID][-][50] После отправки команды задержки устройство будет не реагировать на какие-либо действия в течение указанного времени. Не настраивайте слишком длительные задержки.

	Устройство запоминает текущее значение PC и страницу каждые 30 секунд, и в момент запоминания устройства будет небольшая задержка в ответе.

	Переход на текущую страницу может быть управляем с внешнего MIDI IN, команда соответствует CC 20 XX, где XX - номер страницы для перехода, обратите внимание, что XX должен быть в пределах текущего диапазона страниц устройства.
