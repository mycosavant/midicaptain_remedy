-------------第一部分：-------------

如果您想要定制每一个预设的具体功能和预置里面每一个键按下需要执行的内容，您只需要进入到setup目录，其中globalsetup1~10中包含了每一个控制预设除了脚钉之外的设置内容，对应的preset1~10文件夹中包含了对应预置编号下10个脚钉按键的功能配置内容。下面对每一项代表的意义做一个简要说明：（请注意下面仅仅用于说明，具体的更改请到setup目录中修改对应文件[]中的内容,请不改动任何[]之外的内容！请不要随意修改删除USB盘中任何其他文件，包括隐藏文件）

以下内容存在于 /setup/globalsetup1~10 中

CURRENT_PRESET= [1] 当前预设的编号，此项不需更改，在机器本身屏幕上设置菜单里，修改预置编号后等待一秒并重新开机（最新版程序会在按下Home或者Back后自动重启）即可进入不同的预置

PRESET_NAME   = [JAMP] 当前预设的名字，可以随意修改，注意5个字符以内

DARK_FONTS    = [OFF] ON表示主屏幕字体采用深色，反之OFF为浅色

SCREEN_LIGHT  = [80] 屏幕背光亮度，范围为1~100

LED_LIGHT     = [25] 踩钉灯圈亮度，范围1~100，彩灯耗电较大，推荐在使用电池时，尽可能设低

BATTERY_CHARGE= [OFF] 开启或关闭对电池的自动充电功能，推荐在连接外接电源时开启，充电电流可能占用额外的250mA左右

WIRELESS_2.4G = [ON] 开启内置无线模块，如果您没有搭配 MIDI MATE 无线配件使用，则完全没有必要开启，关闭将节省约30mA的电流

WIRELESS_ID   = [9] 和MIDI MATE无线配件配对使用时的配对ID，只要ID相同，配对过程完全自动

MIDI_CHANNEL  = [1] 设置当前预制所有MIDI指令的MIDI通道，每个预制可以拥有不同的MIDI通道（在Geek模式下，脚钉发送的多指令中每条指令都可以有不同的 MIDI 通道）

PC_NUM_START  = [0] 设置PC指令的总体偏移量，0或者1，也就是从0开始，还是从1开始，方便和某些主机对齐编号

MIDI_THROUGH  = [ON] 设置为ON可以将MIDI IN的收到的外部内容实时旁通到 MIDI OUT 出去

POWER_ON_PIC  = [2] 有三张开机画面可选，其中3为好朋友插画师洪旭创作，特此感谢！

POWER_ON_TEXT = [MIDI CAPTAIN] 开机显示的彩虹文字也可以定制，大家可以定制12个字符以内的任意内容，赠送朋友神器

BANK_STEP     = [4] PC指令班组的跨度，可以是1~8的任何一个，当然常见的是3 4 5，小技巧：设为1，同时将下面的 BANK_SEND 选项设为ON，可以实现上下逐一浏览主机音色库

BANK_SEND     = [OFF] 在按完上下翻组后是否立即发送新组的首个PC值，默认关闭

EXP1_CC#      = [7] 表情踏板1对应的cc控制号，范围是0~127

EXP2_CC#      = [9] 表情踏板2对应的cc控制号，范围是0~127

WHEEL_MANUAL  = [ON] 开启或关闭转盘PC控制功能，默认开启，考虑到脚踩控制时，有可能会误触转盘，有此担心可以关闭

TIME_ENGINE   = [OFF] 开启或关闭时间引擎功能

TIME_TRACK    = [1] 选择时间引擎的轨道，每个预制下都会拥有自己专属的10个轨道，10个预制所以一共有100个轨道

LOOP_REPEAT   = [OFF] 在播放时间引擎轨道时是否自动重复，由于重复时间点是从录制时刻开始，而不是第1个按键，所以当您需要利用它制连续播放的loop，可以在录制开始后迅速按下第一个按键动作

WALLPAPER     = [1] 有20张壁纸可选，每个预设可以不同，范围为0~19

WIRELESS_dB   = [6] 无线发射功率设置，默认6为0dB 0:12dBm 1:10dBm 2:9dBm 3:8dBm 4:6dBm 5:3dBm 6:0dBm 7:-2dBm 8:-5dBm 9:-10dBm 10:-15dBm 11:-20dBm 12:-25dBm 13:-30dBm 14:-25dBm

Key_Press_PC_MODE     = [1] 当设置为PC类型时，0代表根据上下翻页页数和物理脚钉的位置自动生成当前PC值，1代表使用完全固定的PC值

Key_Press_PC_FIXED    = [7] 上述使用固定PC值时对应的PC数值设置，范围0~127

Key_Press_CC_NUMBER   = [21] 当脚钉设置为发送cc指令时，对应的cc编号，范围0~127

Key_Press_CC_VALUE    = [127] 当脚钉设置为发送cc指令时，对应cc编号下的cc数值，范围0~127

Key_Press_CC_TOGGLE   = [0] 当脚钉设置为发送cc指令时，cc数值是否自动翻转，0为关闭自动翻转，1为开启自动翻转，翻转的意思就是在“0，设置值”之间跳变

2.0版本更新了机器出厂的10个预置的基本配置内容，下面做一个简要说明：

预置1： JAMP
初衷为适配软件效果器JamUp。ABCD为PC音色列表控制，上下翻页为4个一组，上排CC值分配为 21，22，23，24，可以被软件重新学习和映射效果器模块的开关。

预置2：BIAS
初衷为适配软件效果器BIAS。和预置1相同，只是上排CC4个按键都带有自动toggle功能，值得一提的是，这个并不是BIAS软性所必须的。方便更广义的为其它设备或软件使用。

预置3：AXFX
为AXE FX系列设计的一种预置模式，ABCD和上下翻页组成PC控制，切整个音色库列表（前128个），上排CC用于不同的模块开关和调音表控制等

预置4：KMPA
为KPA效果器设计的一种预置模式，ABCD4和上下翻页组成PC控制，4对应E，切整个音色库列表（前128个），上排CC（前3个）用于不同的模块开关和调音表控制等

预置5：PC3AM
为只有三个脚钉的设备且PC编组为3的效果器设计，比如Amphero系列桌面版效果器。ABC配合上下完成PC组控制。1234和D都可以用于CC指令控制不同的槽位开关（A1 A2 A3），调音（4），tap（D）

预置6：PC8
完全音色PC列表控制，每组8个音色排列，不管效果器实际音色如何分组，该预置将音色每8个一组排列完前128个共16个分组，可以做到一脚在8个不同音色之间切换

预置7：ALLCC
完全CC控制，适合很多提供了完善的CC控制功能的主机，只需要CC命令就可以配合主机或者软件，比如Ableton live等，实现足够多的控制意图。

预置8：PC+NT
Note指令直接可以控制大多数合成器和软件发声，也可以作为midi控制使用，可以结合CC和PC的控制，实现更有意思的控制功能。

预置9：AxeSN
为AXE FX系列设计的第二种控制模式，这个模式中更多加入了对Scene模式的应用，ABCD被用来做Scene的控制，对应每个音色的0-3快照，1234分别配置为调音表，delay开关，混响开关，tap 功能，上下配置为逐一翻音色列表

预置10：Quadx
为Quad Cortex DSP设计的一种预置模式，连同主机本身实现一种混合控制配置，MIDI Captain上的1234和ABCD用于当前音色的8个Scenes切换（通常来说Scene为场景或者快照，相比切音色列表，切Scene往往具有更快更直接的音色切换响应速度），上下按键分别配置为调音表和tap打点。本机上的脚钉可以在本机上设置当前音色的内置模块的切入切出，同时本机上的PC program change任然不变。这个配置可以极大的发挥效果器的精细全面的参数控制和演奏灵活性，被众多乐手朋友推崇。


-------------第二部分：-------------

关于 Geek MODE 极客模式
绝大多数的需求都可以在常规模式中得到满足，但是如果需要一键发送多条MIDI指令，并且需要为每条指令配置特定的MIDI通道，那您就可以使用MIDI极客模式。

进入极客模式：按住2号按键开机并保持1秒以上再放开。可以很快看到Geek MODE的开机画面。

在极客模式下，每个按键可以发送12条独立的MIDI指令，其中6条在按下时发送，6条在抬起时发送。显示界面直接显示了全部的指令信息，不同按键按下后，显示更新为当前按下的按键所有按顺序发送的指令。此时按下编码器按键，就进入了对当前按键的发送内容编辑模式，转动编码器可以使得光标快速快递移动到需要修改的值上。大部分内容都是所见既所得，极客模式下没有为按键分配上下翻页功能，大多数被控主机都提供了一个CC指令用于翻页，可以替代上下翻页这个用途的设置。极客模式的脚钉灯环自动显示前3条指令的特征（白色：空，青色：PC，紫色：CC，黄色：note），修改完所需内容后，直接再次按下当前亮灯的脚钉，出现短暂的闪烁之后就代表新的配置已经生效。极客模式有自己单独的配置文件，还是按1键开机进入USB模式，在geeksetup目录下有全部的配置文件，其中geeksetup.txt等效与常规模式的globalsetupX.txt, 具体配置内容仍然可以参考第一部分内容。极客模式下的脚钉配置是.dat为后缀的文件，无需改动，全部在主机极客模式运行状态下修改脚钉的发送内容。

正常模式和极客模式完全相互独立，相当于设备开启时选择了完全不同的主程序执行，没有任何交集。
